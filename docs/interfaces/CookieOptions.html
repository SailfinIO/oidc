Interface CookieOptions

Represents the options for serializing cookies.

The CookieOptions interface defines a customizable structure for creating cookies with specific attributes that control their behavior, storage, and security.

CookieOptions

    httpOnly?: boolean;
    maxAge?: number;
    partitioned?: boolean;
    path?: string;
    priority?: Priority;
    sameSite?: boolean | SameSite;
    secure?: boolean;
}

Optionalencode

encode?: (str: string) => string

Specifies a function used to encode a cookie-value.

Since the value of a cookie has a limited character set (and must be a simple string), this function can encode a value into a format suitable for a cookie's value. It should mirror the decoding function used during parsing to ensure consistency.

Default

encodeURIComponent

Example

// Custom encoding function:
const options: CookieOptions = {
  encode: (value) => btoa(value), // Base64 encode the value
};

Optionalexpires

expires?: Date

Specifies the Expires attribute for the cookie as a Date object.

If no expiration is set, the cookie is considered a "session cookie" and will be deleted when the current session ends. If both maxAge and expires are set, maxAge takes precedence.

Example

const options: CookieOptions = {
  expires: new Date(Date.now() + 3600 * 1000), // Cookie expires in 1 hour
};

OptionalmaxAge

maxAge?: number

Specifies the Max-Age attribute in seconds for the cookie.

This defines how long the cookie should persist before it expires. If both maxAge and expires are set, maxAge takes precedence as per the cookie storage model specification. However, some clients may not honor this behavior, so it is advisable to keep them in sync.

Example

const options: CookieOptions = {
  maxAge: 3600, // Cookie expires after 1 hour
};

OptionalsameSite

sameSite?: boolean | SameSite

Specifies the SameSite attribute for the cookie.

Controls how cookies are sent with cross-site requests:

  • true or 'strict': Strict enforcement.
  • 'lax': Lax enforcement.
  • 'none': Explicitly allows cross-site cookies (requires Secure to be true).

Example

const options: CookieOptions = {
  sameSite: SameSite.LAX,
};
